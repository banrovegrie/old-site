It also has important applications in many tasks unrelated to arithmetic, since it can be used with any operations that have the property of associativity:

Raising aaaÔªø to the power of nnnÔªø is expressed naively as multiplication by aaaÔªø done n‚àí1n-1n‚àí1Ôªø times: an=a√óa√óa¬†...aa^n = a\times a\times a\ ...aan=a√óa√óa¬†...aÔªø. However, this approach is not practical for large a or n.

The idea of binary exponentiation is, that we split the work using the binary representation of the exponent.

Let's write¬†n¬†in base 2, for example:

Since the number n has exactly ‚åälogN‚åã+1\lfloor logN\rfloor + 1 ‚åälogN‚åã+1Ôªø digits in base 2, we only need to perform O(logN)O(logN)O(logN)Ôªømultiplications, if we know the powers a1,¬†a2,¬†a4,...alog(N)a^1,\ a^2,\ a^4, ...a^{log(N)}a1,¬†a2,¬†a4,...alog(N)Ôªø.

So we only need to know a fast way to compute those. Luckily this is very easy, since an element in the sequence is just the square of the previous element.

So to get the final answer for 3133^{13}313Ôªø, we only need to multiply three of them (skipping 323^232Ôªø because the corresponding bit in n is not set): 3133^{13}313Ôªø= 6561‚ãÖ81‚ãÖ3 = 1594323

The final complexity of this algorithm is O(logn): we have to compute logn powers of a, and then have to do at most logn multiplications to get the final answer from them.

// binpow(a, b) => a^b
long long binpow(long long a, long long b) 
{
    if (b == 0)
        return 1;
    long long res = binpow(a, b / 2);
    if (b % 2)
        return res * res * a;
    else
        return res * res;
}


long long binpow(long long a, long long b) {
    long long res = 1;
    while (b > 0) {
        if (b & 1)
            res = res * a;
        a = a * a;
        b >>= 1;
    }
    return res;
}
